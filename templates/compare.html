How the Hybrid Works

Architecture of the two-layer prediction system.

ML PIPELINE
  • → Historical CSV data (7 seasons)
  • → Feature engineering (form, ppg, goal diff, h2h)
  • → sklearn Pipeline: Imputer → Scaler → Classifier
  • → Logistic Regression, Random Forest, XGBoost, LightGBM
  • → Stacking ensemble (meta-learner on held-out fold)
  • → Calibrated probabilities via CalibratedClassifierCV
  • → Saved with joblib, loaded at inference time
RULE ENGINE
  • → Recent form (last 5 matches, PPG weighted)
  • → Home/away advantage coefficient
  • → Goals scored vs goals conceded rate
  • → Goal difference trend (5-game vs 10-game)
  • → Head-to-head historical record
  • → Injury placeholder (risk signal)
  • → Outputs a 0–1 confidence score per team
HYBRID BLEND (PRODUCTION)
final_score = (ml_score × 0.65) + (rule_score × 0.35)

The blend weight (65/35) is derived from walk-forward calibration. ML carries more weight because it learns non-linear patterns across 7 seasons. The rule engine anchors the signal with interpretable logic that degrades gracefully when historical data is limited.
